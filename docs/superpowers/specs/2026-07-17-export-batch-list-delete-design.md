# 本地导出批次列表与删除功能设计

日期：2026-07-17

## 目标

将当前只展示最近一次小米笔记导出的统计区域改为本地导出批次列表。用户可以查看任意批次，并删除指定批次在 NoteChange 本地保存的笔记、元数据和附件。删除操作不得修改小米云端或 vivo 中的任何数据。

## 范围

- 展示所有结构完整且能通过 schema 校验的本地导出批次。
- 按导出完成时间倒序排列。
- 每行显示导出时间、笔记数、附件数、需处理数，以及“查看”“删除”操作。
- “查看”打开现有搜索预览弹窗，但数据源改为所选批次。
- “删除”只删除所选批次目录及其本地附件。
- 不提供批量选择或“一键清空”。

## 界面与交互

“迁移批次”区域保留现有标题和“导出小米笔记”按钮，统计大卡片替换为批次列表：

```text
导出时间          笔记    附件    需处理    操作
07-17 10:35       318     19      245       查看  删除
07-16 22:10       305     17      231       查看  删除
```

列表为空时显示“暂无本地导出，可重新导出小米笔记”。完成新导出后，新批次插入列表首行。

删除按钮使用与当前界面协调的危险操作样式。点击后弹出确认框，内容包括批次导出时间、笔记数和附件数，并明确提示：

> 只会删除 NoteChange 本地保存的这批导出文件，不会删除小米云端笔记或 vivo 笔记。

确认按钮文案为“删除本地批次”。删除进行时禁用重复操作；成功后关闭确认框并刷新列表。失败时保留列表项，显示可理解的错误提示。

## 数据与存储

`ExportBundleStore` 增加以下能力：

- `list(): Promise<StoredExportBundle[]>`：扫描导出根目录中的正式批次目录，读取并校验 manifest 与 notes，忽略临时目录和损坏目录，按 `exportedAt` 倒序返回。
- `load(batchId): Promise<StoredExportBundle | null>`：按批次 ID 安全读取指定批次。
- `delete(batchId): Promise<void>`：只允许删除根目录下通过批次 ID 校验的直接子目录，禁止路径穿越；目标不存在时按幂等成功处理。

删除当前 `latest.json` 指向的批次时，存储层将索引更新为剩余批次中最新的一批；没有剩余批次时删除 `latest.json`。这个索引仅用于兼容现有恢复流程，界面始终以批次列表为准，不做自动导航。

目录删除与 `latest.json` 更新必须避免把已删除批次继续暴露为可用状态。删除失败不得更新内存状态或界面列表。

## 运行时与 IPC

`MigrationRuntime` 增加：

- `listExports()`：返回不含正文和附件路径的批次摘要数组。
- `deleteExport(batchId)`：删除指定批次，并在它恰好是当前内存批次时清空当前 bundle、预览缓存与迁移确认状态。

预览查询和详情查询增加 `batchId` 参数。运行时先安全加载对应批次，再执行现有搜索、筛选和详情转换。导入仍只允许用户当前明确选择并确认的批次，不能因为列表刷新而悄悄切换到其他批次。

新增 IPC 通道：

- `notechange:list-exports`
- `notechange:delete-export`

现有预览 IPC 请求扩展 `batchId`，并保持主进程参数校验。渲染进程不直接接触文件系统路径。

## 错误与安全边界

- 删除前必须二次确认。
- `batchId` 必须满足应用生成格式，并在解析后确认目标仍位于导出根目录内。
- 不跟随指向导出根目录外的符号链接。
- 不在日志、错误信息或 IPC 摘要中暴露标题、正文、附件名、Cookie 或令牌。
- 损坏批次不进入普通列表；本功能不负责修复或删除损坏目录。
- 删除只作用于本地文件，不调用小米或 vivo 接口。

## 测试与验收

1. 存储测试覆盖多批次倒序列表、忽略临时/损坏目录、按 ID 加载、删除指定批次、删除最新批次后更新索引、删除最后一批后清理索引，以及路径穿越拒绝。
2. 运行时测试覆盖摘要不泄露正文、查看指定批次、删除当前批次后清空迁移状态、删除其他批次不影响当前选择。
3. IPC 测试覆盖新通道注册、参数转发和错误传播。
4. 界面测试覆盖列表渲染、空状态、查看指定批次、确认删除、取消删除、删除成功刷新、删除失败保留行。
5. 全量测试、类型检查、契约校验和 Electron 构建通过。
6. 手工验证确认删除后批次目录及附件不存在，而小米官网笔记数量和 vivo 数据不发生变化。

## 非目标

- 批量删除或清空全部批次。
- 恢复已删除的本地批次。
- 删除云端笔记。
- 在本功能中修复损坏批次或迁移旧数据格式。
