# 授权后台运维控制台设计

## 目标

将授权服务的管理页面改为适合日常运营的高信息密度控制台，并把桌面应用更新发布变为可在网页内完成的上传、校验和发布流程。

## 使用者与任务

使用者是 NoteChange 的授权服务管理员。页面的首要任务是确认服务是否健康、管理永久激活码、发布 macOS 和 Windows 安装包，以及安全地查看本地授权服务配置状态。

## 信息架构

- 概览：展示授权码数量、启用的平台版本、服务运行状态和 GitHub 更新状态。
- 激活码：生成、查看、解绑和禁用激活码。
- 版本发布：按“上传文件、填写版本、确认发布”三个连续步骤完成更新发布。
- 服务更新：检查并拉取 GitHub `main`，成功后由宝塔进程守护自动重启。
- 系统设置：显示数据目录、密钥、密码哈希和数据库的保存位置与状态，但不显示私钥、公钥内容、密码或哈希值。

## 视觉方向

采用深色左侧导航和浅色工作区。导航使用 `#102235`，页面底色 `#F4F7FA`，工作面 `#FFFFFF`，主操作 `#176B87`，正常状态 `#16705B`，警示 `#B7791F`，错误 `#B83A3A`。系统文字使用平台中文 sans-serif；路径、版本、SHA-512 使用等宽字体。

页面在宽屏使用固定左侧导航和两列概览数据；窄屏收起为横向导航，并将表格保留横向滚动。按钮使用清晰的文本命令，危险操作保持次要样式和确认提示。

## 发布流程

1. 管理员选择目标平台并上传对应安装包。服务端将文件写入授权服务数据目录中的发布目录，限制为安全文件名并拒绝路径穿越。
2. 服务端流式计算 SHA-512，返回可下载的相对文件路径、文件名和摘要；浏览器不需要手工计算哈希。
3. 管理员输入语义化版本号和更新说明，确认后发布。服务端登记发布记录并继续通过 `latest.yml` 或 `latest-mac.yml` 向桌面端提供更新清单。

上传成功但尚未发布的文件会保留在发布目录，方便重新填写版本或发布；发布列表可启用和停用对应平台的当前版本。

## 配置状态与安全

系统设置接口只返回如下元数据：数据目录和各配置文件路径、私钥/公钥是否存在、管理员密码是否已设置、数据库记录数、数据库文件修改时间。它不返回密钥、密码、哈希或会话 Cookie。

所有上传、发布、更新和状态接口都保留现有的管理员 HttpOnly 会话校验。上传请求使用原始二进制流、固定大小上限和固定文件名校验；服务端命令继续使用固定参数的 `spawn` 调用，不接收浏览器提供的命令。

## 验证

- 对上传路径校验、SHA-512、发布记录和状态元数据增加服务端测试。
- 使用临时数据目录验证：创建密码、上传一个小文件、发布 Windows 更新、读取 `latest.yml` 和下载文件。
- 运行 `npm test`、`npm run typecheck`、`npm run build`，并对管理页面作桌面与移动宽度截图检查。
