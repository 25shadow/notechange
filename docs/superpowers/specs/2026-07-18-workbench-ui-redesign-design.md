# 云笔记迁移工作台界面改造设计

## 背景

当前页面使用步骤箭头、迁移确认勾选和“需处理”统计来引导迁移流程。随着本地批次预览、附件查看和登录态复用功能增加，这种流程式界面已经产生重复信息，也让用户误以为每次导入都必须先确认一组警告。

## 目标

- 将顶部步骤箭头改为最近操作日志。
- 将小米和 vivo 区域收敛为登录状态管理，不承担迁移流程说明。
- 通过“导出笔记”弹窗选择导出平台。
- 在批次查看页面增加“导入”入口，通过弹窗选择目标平台。
- 移除“需处理”列、警告数量展示和“我已核对目标账号和迁移数量”勾选框。
- 所有用户可见文案使用中文。

## 非目标

- 不改变小米、vivo 的接口契约。
- 不重新设计本地批次文件格式。
- 不在本次改造中扩展 vivo 导出或小米导入能力；未实现的平台方向需要在弹窗中明确显示为暂不可用，而不是静默失败。
- 不删除本地批次或云端笔记的既有行为。

## 页面结构

```text
标题栏：云笔记迁移工作台                         最近操作
                                                   最近状态列表
账号连接：
  小米云笔记卡片                                  vivo 原子笔记卡片
  登录小米 / 已登录                               登录 vivo / 已登录

本地笔记批次：                                    [导出笔记]
  导出时间 | 笔记数 | 附件数 | 操作（查看、删除）
```

### 最近操作日志

日志只保存并展示最近操作状态，不展示接口、请求参数或堆栈。每条记录包含时间、中文状态和成功/失败视觉状态。至少覆盖登录、导出开始、导出完成、批次删除、导入开始、导入完成和失败。

### 登录卡片

每个平台显示名称、当前账号标签和状态：

- 未登录：显示“未登录”和“登录小米”或“登录 vivo”。
- 已登录：显示绿色“已登录”，仍允许重新打开登录流程。
- 连接或登录异常：显示中文错误信息，并写入日志。

### 导出选择

“导出笔记”按钮打开平台选择弹窗：

- “导出小米笔记”：当小米已登录时可用。
- “导出 vivo 笔记”：当前显示“暂未支持”并禁用，保留后续扩展位置。

选择后关闭弹窗，开始导出并在日志中记录状态。导出完成后刷新本地批次列表。

### 批次列表

保留导出时间、笔记数量、附件数量和“查看”“删除”操作。删除需要保留现有删除确认弹窗；移除“需处理”列和所有 warningCount 相关 UI 展示，但不删除底层数据中的真实错误信息。

### 预览和导入

预览弹窗左侧增加“导入”按钮。点击后打开目标平台选择弹窗：

- “导入到 vivo”：vivo 已登录时可用，并沿用现有导入流程。
- “导入到小米”：当前显示“暂未支持”并禁用。

不再显示或要求迁移确认复选框。导入前仅根据目标平台登录状态决定按钮可用性；未登录时提示“请先登录 vivo”。导入过程和结果写入最近操作日志。

## 状态与错误处理

- 选择不可用平台时不发起 IPC 请求。
- 导出/导入进行中，相关按钮显示处理中状态并禁用重复提交。
- 错误同时显示短中文提示和日志记录；详细错误继续保留在主进程日志中。
- 最近操作日志只保留固定数量的最新记录，避免页面无限增长。

## 验收标准

1. 页面顶部没有步骤箭头。
2. 页面可看到中文“最近操作”区域，且登录、导出、删除、导入状态会更新。
3. 小米/vivo 卡片只承担登录，登录后明确显示“已登录”。
4. “导出笔记”弹窗可以选择平台；未支持方向不会误触发请求。
5. 批次列表没有“需处理”列。
6. 预览左侧存在“导入”按钮，点击后可以选择目标平台。
7. 页面不再出现“我已核对目标账号和迁移数量”复选框。
8. 现有小米导出、批次查看、删除、vivo 导入测试继续通过。
